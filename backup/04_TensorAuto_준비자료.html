산출물 — 자율주행 제어 친화적 모터 특성 요약

"모터가 예측대로 움직이지 않으면, 알고리즘의 예측 모델이 무너진다."

배경 자율주행 소프트웨어의 진짜 적

Level 4 자율주행은 수 밀리초 단위로 제어 명령을 모터에 전달한다. 이 명령이 물리적 차량 거동으로 오차 없이 구현되어야 AI가 작동한다.

모터의 출력이 강한가가 아니라,
모터가 얼마나 정직하게 제어 명령을 따르는가 — 이것이 핵심이다.

핵심 특성 eFlow AFPM — 자율주행 제어 친화적 특성 3가지
① Zero Cogging — 저속 정밀 제어 정확도 향상
문제: 일반 모터의 코깅 토크(Cogging Torque)
      → 자석과 슬롯 간 자기적 끌림 현상
      → 저속 구간에서 토크 리플(Torque Ripple) 발생
      → AI 궤적 계산과 실제 차량 움직임 간 오차 발생

eFlow 해법: SMC(연자성 복합소재) 스테이터 적용
           → 3D 자속 경로 최적화
           → 코깅 토크 최소화
           → FOC(Field Oriented Control) 벡터 제어로 토크 리플 보상

결과: 도심 교차로·저속 정밀 주차 구간에서
      AI 제어 명령 = 차량 물리 거동 일치율 극대화
② Zero Derating — 알고리즘 예측 모델 안정성 확보
문제: 일반 모터의 열 스로틀링(Thermal Derating)
      → 고속·고부하 주행 중 모터 발열
      → 출력 강제 저하 발생
      → AI가 예측한 가속·감속 프로파일과 실제 출력 불일치

eFlow 해법: 밀폐형 오일 냉각 구조
           → 스테이터·로터 분리 냉각
           → 150°C 연속 구동 가능 (기존 모터 120°C 대비)
           → 고부하 구간에서도 출력 편차 없음

결과: 어떤 주행 조건에서도 AI 예측 모델이
      동일한 모터 출력 응답을 기대할 수 있음
③ Low Noise · Vibration — 센서 간섭 원천 차단
문제: 모터 진동·소음의 센서 간섭
      → 차체에 탑재된 LiDAR·레이더 센서
      → 모터 진동이 센서 측정값에 노이즈로 유입
      → 물체 인식 정확도 저하

eFlow 해법: AFPM 팬케이크 구조 특성상 저진동·저소음
           → 크로스 롤러 베어링 적용, 축방향·반경방향 하중 동시 지지
           → NVH(소음·진동·하시니스) 설계 최적화

결과: LiDAR·레이더 센서 측정 정확도 유지
      → AI 인식 모델 입력 데이터 품질 보호
